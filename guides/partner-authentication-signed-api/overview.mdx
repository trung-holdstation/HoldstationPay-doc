---

For signed API, partners need to use Ed25519 signature-based authentication instead of JWT. Each request body must be cryptographically signed using an Ed25519 private key held by the partner. Holdstation Pay verifies the signature on the server side using the partner's registered public key (`verify_key`). This ensures both the authenticity and integrity of every request, preventing tampering or replay attacks.

<Note>
  Signed API authentication is required in addition to the `Partner-App-Key` header — both must be present on every signed request.
</Note>

---

## Required Headers

Each signed request must include:

| Header | Description |
|---|---|
| `Partner-App-Key` | The partner's app key (same as used for other integrations) |
| `X-HSPay-Signature` | Base64-encoded Ed25519 signature of the raw JSON request body |

The signature is computed over the **raw JSON body** (byte-for-byte) using the partner's Ed25519 private key. The server verifies using the partner's `verify_key` (Ed25519 public key in base64).
