---
title: Create Withdrawal
openapi: POST /partners/orders/withdrawal
---

Creates a new offramp order for a user to sell tokens and receive VND via bank transfer. See [Offramp Flow](/guides/offramp-flow) for the full integration guide.

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `amount` | string | ‚úÖ | Amount to withdraw (in USD or VND depending on `currency`) |
| `currency` | string | ‚úÖ | Currency of the amount: `vnd` or `usd` |
| `chain_id` | integer | ‚úÖ | Blockchain network ID ‚Äî see [List Tokens](/api-reference/tokens/list-tokens) |
| `token_address` | string | ‚úÖ | Token contract address ‚Äî see [List Tokens](/api-reference/tokens/list-tokens) |
| `payment_info` | object | ‚úÖ | User's bank account to receive VND ‚Äî see [PaymentInfo](#paymentinfo) |
| `callback` | string | ‚úÖ | HTTPS URL to receive order status webhook notifications |
| `reference_no` | string | ‚úÖ | Partner's own reference number for tracking |
| `buffer` | number | ‚úÖ | Rate buffer override (fractional). Effective buffer = max(override, configured). Example: `0.005` = 0.5% |

### PaymentInfo

| Field | Type | Required | Description |
|---|---|---|---|
| `bank_id` | integer | ‚úÖ | Bank ID ‚Äî see [List Banks](/api-reference/banks/list-banks) |
| `full_name` | string | ‚úÖ | Full name of the account holder |
| `account_type` | integer | ‚úÖ | `1` = Bank account number, `2` = ATM card number |
| `account_number` | string | ‚úÖ | Bank account number |

### Order Object

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique order UUID |
| `state` | integer | `1`=New, `2`=Processing, `3`=Completed, `4`=Failed, `5`=Closed |
| `processing_state` | integer | `20`=CryptoPending, `21`=CryptoConfirmed, `22`=CryptoFailed, `23`=FiatPending, `24`=FiatConfirmed, `25`=FiatFailed |
| `order_type` | integer | `2` = Withdrawal |
| `amount` | string | Input amount in USD |
| `outcome` | string | VND amount the user will receive after fees |
| `rate` | string | Exchange rate applied to this order |
| `original_rate` | string | Exchange rate before buffer adjustment |
| `currency` | string | Currency of the amount (`vnd` or `usd`) |
| `total_fee_vnd` | number | Total fee in VND |
| `fees` | object | Fee breakdown ‚Äî see [Fees](#fees) |
| `token_address` | string | Token contract address |
| `chain_id` | integer | Blockchain network ID |
| `recipient` | string | Holdstation Pay's wallet address the user must send tokens to |
| `reference_no` | string | Partner's reference number |
| `external_id` | string | External order ID from payment provider |
| `partner_id` | string | Partner identifier |
| `user_id` | string | User identifier |
| `code` | integer | Internal status code from payment provider |
| `provider` | integer | Payment provider identifier |
| `processing_time` | integer | Time taken to process in milliseconds |
| `client_ip` | string | IP address of the client that created the order |
| `callback` | string | Webhook callback URL |
| `pay_data` | object | `null` for withdrawal orders |
| `payment_info` | object | Bank account details the VND will be sent to ‚Äî see [PaymentInfo (Response)](#paymentinfo-response) |
| `error` | object | Error details if failed ‚Äî see [OrderError](#ordererror) |
| `created_at` | object | Order creation timestamp ‚Äî `{ seconds, nanos }` |
| `updated_at` | object | Last update timestamp ‚Äî `{ seconds, nanos }` |
| `expired_at` | object | Order expiry timestamp ‚Äî `{ seconds, nanos }` |

### PaymentInfo (Response)

The bank account the VND payout will be sent to:

| Field | Type | Description |
|---|---|---|
| `bank_id` | integer | Bank ID |
| `full_name` | string | Account holder name |
| `account_type` | integer | `1` = Bank account number, `2` = ATM card number |
| `account_number` | string | Bank account number |

### Fees

| Field | Type | Description |
|---|---|---|
| `service_fee` | number | Service fee in VND |
| `min_fee` | number | Minimum fee applied in VND |

### OrderError

Populated when `state = 4` (Failed):

| Field | Type | Description |
|---|---|---|
| `error_code` | string | Machine-readable error code |
| `error_message` | string | Human-readable error description |
| `details` | string | Additional error details |
| `retryable` | boolean | Whether the operation can be retried |
| `source` | integer | Internal error source identifier |
| `timestamp` | string | When the error occurred |
