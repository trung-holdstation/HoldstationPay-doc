---
title: Create Deposit
openapi: POST /partners/orders/deposit
---

Creates a new onramp order for a user to buy tokens by depositing VND via bank transfer. See [Onramp Flow](/guides/onramp-flow) for the full integration guide.

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `amount` | string | ✅ | Amount to deposit (in VND or USD depending on `currency`) |
| `currency` | string | ✅ | Currency of the amount: `vnd` or `usd` |
| `chain_id` | integer | ✅ | Blockchain network ID — see [List Tokens](/api-reference/tokens/list-tokens) |
| `token_address` | string | ✅ | Token contract address — see [List Tokens](/api-reference/tokens/list-tokens) |
| `recipient` | string | ✅ | User's wallet address to receive the tokens |
| `callback` | string | ✅ | HTTPS URL to receive order status webhook notifications |
| `reference_no` | string | ✅ | Partner's own reference number for tracking |
| `buffer` | number | ✅ | Rate buffer override (fractional). Effective buffer = max(override, configured). Example: `0.005` = 0.5% |

### Order Object

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique order UUID |
| `state` | integer | `1`=New, `2`=Processing, `3`=Completed, `4`=Failed, `5`=Closed |
| `processing_state` | integer | `10`=FiatPending, `11`=FiatConfirmed, `12`=FiatFailed, `13`=CryptoPending, `14`=CryptoConfirmed, `15`=CryptoFailed, `16`=FiatWaitingForApproval |
| `order_type` | integer | `1` = Deposit |
| `amount` | string | Input amount in VND |
| `outcome` | string | Token amount the user will receive after fees |
| `rate` | string | Exchange rate applied to this order |
| `original_rate` | string | Exchange rate before buffer adjustment |
| `currency` | string | Currency of the amount (`vnd` or `usd`) |
| `total_fee_vnd` | number | Total fee in VND |
| `fees` | object | Fee breakdown — see [Fees](#fees) |
| `token_address` | string | Token contract address |
| `chain_id` | integer | Blockchain network ID |
| `recipient` | string | User's wallet address to receive the tokens |
| `reference_no` | string | Partner's reference number |
| `external_id` | string | External order ID from payment provider |
| `partner_id` | string | Partner identifier |
| `user_id` | string | User identifier |
| `code` | integer | Internal status code from payment provider |
| `provider` | integer | Payment provider identifier |
| `processing_time` | integer | Time taken to process in milliseconds |
| `client_ip` | string | IP address of the client that created the order |
| `callback` | string | Webhook callback URL |
| `pay_data` | object | Bank payment details to display to the user — see [PayData](#paydata) |
| `payment_info` | object | `null` for deposit orders |
| `error` | object | Error details if failed — see [OrderError](#ordererror) |
| `created_at` | object | Order creation timestamp — `{ seconds, nanos }` |
| `updated_at` | object | Last update timestamp — `{ seconds, nanos }` |
| `expired_at` | object | Order expiry timestamp — `{ seconds, nanos }` |

### PayData

Bank payment details the partner should display to the user:

| Field | Type | Description |
|---|---|---|
| `bank_account_name` | string | Beneficiary name on the bank account |
| `bank_account_number` | string | Bank account number to transfer to |
| `bank_bin_code` | string | Bank BIN/routing code |
| `bank_name` | string | Full bank name |
| `bank_short_code` | string | Bank short code |
| `payment_link` | string | Deep-link URL to open the user's banking app |
| `qr_code` | string | VietQR code string |
| `qr_image` | string | Base64-encoded QR code image |
| `external_id` | string | Payment provider's internal reference ID |

### Fees

| Field | Type | Description |
|---|---|---|
| `service_fee` | number | Service fee in VND |
| `min_fee` | number | Minimum fee applied in VND |

### OrderError

Populated when `state = 4` (Failed):

| Field | Type | Description |
|---|---|---|
| `error_code` | string | Machine-readable error code |
| `error_message` | string | Human-readable error description |
| `details` | string | Additional error details |
| `retryable` | boolean | Whether the operation can be retried |
| `source` | integer | Internal error source identifier |
| `timestamp` | string | When the error occurred |
